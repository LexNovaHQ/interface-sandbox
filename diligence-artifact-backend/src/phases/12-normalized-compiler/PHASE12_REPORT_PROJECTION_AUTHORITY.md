# Phase 12 Report Projection Authority

## Governing rule

**Upstream phases decide. Phase 12 arranges.**

Phase 12 prepares the report artifacts. It does not repeat, repair, reinterpret or supplement analysis performed upstream.

## Permitted work

Phase 12 may perform only:

- exact compiler admission;
- deterministic source-path selection from canonical material profile artifacts;
- FDR field-ID binding;
- mounted Registry Key code-to-`normalized_name` lookup;
- fixed section and subsection placement;
- fixed ordering and grouping;
- deterministic counts over upstream-owned values;
- custody, schema, identity and completeness validation;
- report manifest and renderer-handoff assembly.

## Forbidden work

Phase 12 must not create or change:

- legal findings;
- exposure identities;
- severity, Pain Tier, Pain Category or Pain Depth;
- priority;
- review route;
- remediation;
- diligence or clarification questions;
- limitations;
- handoff blocks;
- upstream status;
- legal or material conclusions through regex or heuristic logic.

Phase 12 must not review raw source evidence or use forensic artifacts as substantive report inputs. It must not generate prose that changes the meaning of an upstream field.

## Public terminology

Client-facing report labels use:

| Internal identifier | Public report label |
|---|---|
| `primary_domain_package` | Primary Sector |
| `domain_derivation_profile` | Sector Classification Profile |

| `regulatory_overlays` | Regulatory Context Overlays |
| `domain_control_obligation_profile` | Sector-Specific Control Obligations |

Internal artifact names and backend field names remain unchanged. No backend-wide `domain` to `sector` migration is authorized.

## Future direct material-profile input boundary

`PHASE12_DIRECT_PROFILE_ARTIFACTS_NO_PHASE2G`

The rebuilt Phase 12 reads canonical material phase-profile artifacts directly through runtime artifact access. Phase 2G has no routing, bucket or packet role in Phase 12.

The rebuilt Phase 12 must not read:

- `phase_routing_manifest`;
- `phase_route_runtime_packet`;
- Phase 2G bucket maps;
- Phase 2G-routed lossless evidence;
- raw source evidence;
- forensic profiles as substantive inputs;
- Phase 11 internal semantic, reinvestigation, checkpoint or dispatch artifacts.

The exact direct material-profile allowlist is frozen with the report ownership matrix in the next Phase 12 change order.

## Atomic cutover

This authority does not remove the current Phase 2G-based compiler loader. The later compiler rebuild must install direct profile-artifact reads and remove the existing Phase 2G dependency atomically. Phase 12 must never read Phase 2G and direct phase artifacts simultaneously.

## Output architecture

The rebuilt compiler will emit one canonical artifact per final report section. The renderer may format those artifacts, but may not merge semantic sections or create new conclusions.
