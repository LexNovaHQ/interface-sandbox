# 00_TERMINAL_RECEIPT_RULES_INTEGRATED
## Agent 3 Target Feature Terminal Rules — Backend Mode

This file governs `agent_3_target_feature` only.

## Backend rule

In backend execution, Agent 3 must not emit a terminal receipt inside the model response.

The backend runner saves artifacts and advances phases. The model response must remain strict JSON only.

Forbidden in model response:

- markdown
- prose receipt
- same-chat next-agent command
- `<phase_output>` blocks
- checkpoint lines
- report text
- final output handoff
- renderer payload
- legal-risk prose
- privacy findings
- registry findings
- audit dumps

## M7 backend response

For `M7_TARGET_PROFILE`, return only:

```json
{
  "target_profile": {},
  "target_profile_forensics": {}
}
```

No other top-level keys.

## M8 backend response

For `M8_TARGET_FEATURE_PROFILE`, return only:

```json
{
  "target_feature_profile": {},
  "target_feature_profile_forensics": {}
}
```

No other top-level keys.

## Phase failure rule

If M7 cannot produce valid outputs, return the required JSON shape with controlled limitation/repair details inside `target_profile_forensics`; do not run or reference M8 as executed.

If M8 cannot produce valid outputs, return the required JSON shape with controlled limitation/repair details inside `target_feature_profile_forensics`; preserve M7 as prior saved state and do not reference M10 as executed.

## Human receipt rule

Human-readable receipts are generated by the backend or reviewer UI, not by the model during backend execution.
